## 🛠 2. Strict Coding Standards

### A. Naming Conventions

- **Files & Folders**: `camelCase` (e.g., `reportLogic.ts`, `userProfile.tsx`)
- **Components**: `PascalCase` (e.g., `SubmitButton`, `MapContainer`)
- **Variables/Functions**: `camelCase` (e.g., `fetchUserData`)
- **Constants**: `UPPER_SNAKE_CASE` (e.g., `MAX_IMAGE_SIZE`)
- **Interfaces/Types**: `PascalCase` (e.g., `UserInterface`)

### B. Clean Code Rules

- **DRY (Don't Repeat Yourself)**: Extract repeated logic/JSX into reusable Hooks or Components
- **No Hardcoding**:
  - **Colors/Sizes**: Must strictly come from `src/constants/theme.ts`
  - **Text/Strings**: Must use i18n. Use `t('welcome_message')`
- **Separation of Concerns**: Components = UI; Hooks = Logic; Services = API
- **Type Safety**: Strict TypeScript for Frontend; Pydantic for Backend. No `any` types allowed

### C. Comments & Documentation

- Add **JSDoc** (TS) or **Docstrings** (Python) for complex functions
- Comment **"Why"** logic exists, not **"What"** it does

## ⚛️ 3. Frontend Guidelines (React/React Native)

### Component Structure Skeleton (Strict Order)

Every functional component must follow this order:

```typescript
// 1. Imports (External -> Local Components -> Hooks -> Constants)
import React, { useState, useCallback, useMemo } from 'react';
import { useTranslation } from 'react-i18next';
import { CustomButton } from '../components/CustomButton';
import { useReportLogic } from '../hooks/useReportLogic';
import { COLORS, SPACING } from '../constants/theme';

// 2. Interfaces
interface Props {
  title: string;
}

// 3. Component Definition
export const MyComponent: React.FC<Props> = ({ title }) => {
  const { t } = useTranslation();
  const { submitData, isLoading } = useReportLogic();
  const [localState, setLocalState] = useState(false);

  // Derived State (useMemo)
  const buttonColor = useMemo(
    () => localState ? COLORS.secondary : COLORS.accent,
    [localState]
  );

  // Event Handlers (useCallback - ALWAYS)
  const handlePress = useCallback(() => {
    submitData();
  }, [submitData]);

  return (
    <View style={styles.container}>
      <Text style={styles.text}>{t('report_title')}</Text>
      <CustomButton onPress={handlePress} color={buttonColor} />
    </View>
  );
};

// 4. Styles
const styles = StyleSheet.create({
  container: {
    padding: SPACING.medium,
    backgroundColor: COLORS.background
  },
  text: {
    color: COLORS.primary
  }
});
```

## 🐍 4. Backend Guidelines (Python FastAPI)

- **Thin Routers**: Routers only handle validation and calling a Service
- **Service Layer**: All business logic (OpenCV, AI inference, DB transactions) stays in `services/`
- **Error Handling**: Use a global exception handler. Raise `HTTPException` with clear status codes

---

## 🌍 5. Internationalization (i18n)

- All strings in `src/i18n/locales/en.json` and `ms.json`
- Keys must be hierarchical: `auth.login_button`
