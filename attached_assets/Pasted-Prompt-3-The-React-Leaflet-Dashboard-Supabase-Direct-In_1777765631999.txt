Prompt 3: The React Leaflet Dashboard (Supabase Direct & Interactive)
Task: Build the frontend Web Dashboard UI for "JalanGuard".

Context: We are using React (with Tailwind CSS) and React Leaflet (react-leaflet, leaflet). We are NOT using a custom backend API. We will fetch data directly from our Supabase PostgreSQL database using @supabase/supabase-js.

Requirements:

1. Global Aesthetic & Layout:

Use a dark-theme aesthetic (Navy background #0b0f19, Amber/Gold accents).

Remove any hardcoded or static "Issue Details" side cards from the layout (like the old #JG-2841 mock). The map should take up the full screen.

2. The View Toggle (CRITICAL FIX):

There must be two buttons in the top-right or bottom-right corner to toggle between "Heatmap" and "Pins" views.

Fix the logic: Use a React state variable (e.g., const [mapView, setMapView] = useState('heatmap')). These buttons MUST correctly shift the view by conditionally rendering either the Heatmap layer or the Pins layer inside the React Leaflet map.

3. Data Fetching & Layers:

Heatmap Layer: Fetch data from supabase.from('state_heatmap_stats').select('*'). Parse the geojson column. Render this using React Leaflet's <GeoJSON/> component. Color the polygons dynamically based on the high_severity_ratio (e.g., Red > 50%, Orange > 20%, Green otherwise). Add a tooltip showing state_name and total_reports.

Pins Layer: Fetch data from supabase.from('hazards').select('*').eq('status', 'active'). Render these as <Marker/> components at their respective latitude and longitude.

4. Dynamic Hazard Card:

Create a React state const [selectedHazard, setSelectedHazard] = useState(null).

When a user clicks a Marker in the Pins view, update selectedHazard with that hazard's data.

ONLY when selectedHazard is not null, render an absolutely positioned card overlay over the map showing the specific hazard's details (Severity, Lat/Lng, Image, etc.). Include a close button (x) that sets selectedHazard back to null.

5. Fallback UI & Error Boundary:

Wrap the map component in a React Error Boundary and handle loading states.

If the map fails to load, or data fetching throws an error, DO NOT show a blank screen.

Render a highly styled fallback component: Use a dark-themed SVG background pattern of intersecting grid lines and abstract map nodes (a "blueprint" look). Display a clear error message ("Map data currently unavailable") and a prominent "Retry Connection" button.

Goal: A flawlessly interactive, dark-mode React Leaflet dashboard that successfully toggles between a data-driven Heatmap and individual interactive Pins, complete with a professional fallback UI.
